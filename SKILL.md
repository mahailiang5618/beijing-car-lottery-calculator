# 北京小客车家庭摇号积分计算

帮助用户计算北京市小客车指标家庭申请的家庭总积分，包含每位家庭成员的个人积分明细和最终家庭总积分。

## 积分规则速查

### 个人积分 = 基础积分 + 阶梯积分

| 成员角色 | 基础积分 |
|---------|---------|
| 主申请人 | 2分 |
| 其他家庭成员（配偶、父母、子女等） | 每人1分 |

**阶梯积分**：从2014年1月开始累计参与摇号（每2个月一期，每年6期），每累计24次未中签 → +1分阶梯积分（即连续参与约4年+1分）。

### 家庭总积分公式

- **含配偶**：`[(主申请人积分 + 配偶积分) × 2 + 其他成员积分之和] × 代际数`
- **不含配偶**：`[主申请人积分 + 其他成员积分之和] × 代际数`

### 代际数

| 代际 | 系数 | 示例 |
|------|------|------|
| 1代 | ×1 | 仅夫妻两人 |
| 2代 | ×2 | 父母+子女 |
| 3代 | ×3 | 祖父母+父母+子女 |

## 使用流程

### 第1步：收集家庭成员信息

向用户逐步询问以下信息：

1. **主申请人**：从哪一年哪一月开始参与摇号？（或累计未中签次数）
2. **配偶**：是否有配偶一起申请？配偶从何时开始摇号？
3. **其他家庭成员**：还有哪些成员参与？（父母、子女等）各自从何时开始摇号？
4. **代际关系**：家庭成员涵盖几代人？

引导话术：
- "请问主申请人是从哪一年开始参加摇号的？（如不确定月份，默认按1月计算）"
- "家庭中是否有配偶一起申请？配偶从哪年开始摇号？"
- "还有哪些家庭成员参与？他们的关系和开始摇号的时间分别是？"
- "您的家庭成员涵盖几代人？（夫妻=1代，父母+子女=2代，祖父母+父母+子女=3代）"

如果用户不确定月份，默认使用1月。如果用户直接提供了累计未中签次数，可以跳过时间计算。

### 第2步：执行计算

收集完信息后，构造 JSON 并调用计算脚本：

```bash
python3 ~/.claude/skills/beijing-car-lottery/scripts/calculate_family_score.py --json '<JSON数据>'
```

**JSON 输入格式：**

```json
{
  "primary_applicant": {
    "name": "姓名",
    "start_year": 2014,
    "start_month": 1
  },
  "spouse": {
    "name": "配偶姓名",
    "start_year": 2016,
    "start_month": 1
  },
  "other_members": [
    {
      "name": "成员姓名",
      "relationship": "父亲",
      "start_year": 2018,
      "start_month": 1
    }
  ],
  "generations": 2,
  "reference_date": "2026-05"
}
```

字段说明：
- `start_year` / `start_month`：开始参与摇号的年月
- `unsuccessful_rounds`：可选，直接指定累计未中签次数（提供此字段则忽略 start_year/start_month）
- `spouse`：无配偶时设为 `null`
- `other_members`：无其他成员时设为 `[]`
- `generations`：1/2/3
- `reference_date`：可选，格式 YYYY-MM，默认当前月

### 第3步：展示结果

直接将脚本输出的 Markdown 报告展示给用户。

## 注意事项

- 摇号从2014年1月开始计算，之前的参与不计入
- 每2个月一期（1月、3月、5月、7月、9月、11月），每年6期
- 阶梯积分 = floor(累计未中签次数 / 24)
- 如果用户不确定信息，建议其查询"北京市小客车指标调控管理信息系统"(https://xkczb.beijing.gov.cn)
- 本工具仅计算家庭积分，不直接预测中签概率
